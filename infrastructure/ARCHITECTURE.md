## Technology Stack

### Frontend
- **Framework**: React + Vite
- **Styling**: TailwindCSS
- **Build**: Multi-stage Docker (node вҶ’ nginx)
- **Server**: nginx (static files + API proxy)

### Backend
- **Framework**: FastAPI (Python 3.11)
- **ORM**: SQLModel
- **Database**: PostgreSQL
- **Server**: Uvicorn

### Infrastructure
- **Cloud**: AWS
- **Container Orchestration**: EKS (Kubernetes)
- **Compute**: Fargate (serverless)
- **Container Registry**: ECR
- **IaC**: Terraform
- **Networking**: VPC with public/private subnets

## Fargate vs EC2 Nodes

We use **Fargate** (serverless) instead of EC2 nodes:

| Aspect | Fargate | EC2 Nodes |
|--------|---------|-----------|
| **Cost Model** | Pay per pod/second | Pay per node/hour |
| **Scaling** | Automatic | Manual/Cluster Autoscaler |
| **Management** | Zero | Patching, updates |
| **Idle Cost** | None (scale to 0) | Always running |
| **Best For** | Variable workloads | Steady workloads |

## Security

### Network Security
- Private subnets for pods
- NAT Gateway for outbound traffic
- Security groups for access control

### Secrets Management
- Kubernetes Secrets for sensitive data
- Environment variables injected at runtime
- Secrets stored in `infrastructure/secrets/.env` (gitignored)

### IAM
- Least privilege IAM roles
- Separate roles for cluster and pods
- OIDC provider for service accounts

## Scaling Strategy

### Current Setup
- Frontend: 2 replicas
- Backend: 2 replicas
- PostgreSQL: 1 replica (stateful)

### Scale to Zero (Cost Saving)
```bash
kubectl scale deployment --all --replicas=0 -n tinder-languages
```

### Scale Up
```bash
kubectl scale deployment/frontend --replicas=2 -n tinder-languages
kubectl scale deployment/backend --replicas=2 -n tinder-languages
```

## Disaster Recovery

### Backup Strategy
- PostgreSQL: PersistentVolumeClaim (EBS)
- Application: Stateless, rebuild from ECR images
- Infrastructure: Terraform state

### Recovery Steps
1. Run `terraform apply` to recreate infrastructure
2. Run `./scripts/deploy.sh k8s` to deploy application
3. Restore database from backup if needed
